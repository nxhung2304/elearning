# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
make setup          # Install gems and prepare the database
make dev            # Start the development server (Procfile.dev via bin/dev)
make test           # Run the full test suite (bin/rails test)
make lint           # Run RuboCop (rubocop-rails-omakase style)
make security       # Run Brakeman static analysis
make audit          # Run bundler-audit for vulnerable gems
bin/ci              # Run all CI steps in sequence (lint, security, audit, tests, seeds)
```

Run a single test file:
```bash
bin/rails test test/models/user_test.rb
bin/rails test test/controllers/users_controller_test.rb
```

Code generators (via Makefile wrappers):
```bash
make gen-model      NAME=Course FIELDS="title:string price:decimal"
make gen-controller NAME=Courses ACTIONS="index show"
make gen-resource   NAME=Course FIELDS="title:string"
make gen-migration  NAME=AddStatusToUsers FIELDS="status:integer"
```

## Architecture

Rails 8.1 app với PostgreSQL, Solid Queue + Solid Cache + Solid Cable, Hotwire (Turbo + Stimulus). Asset pipeline dùng Propshaft + Importmap (không có Node/Webpack).

**Auth:** Devise (session-based). Tất cả routes ngoài Devise paths đều require authentication. `ApplicationController` gates access.

**Authorization:** CanCanCan — xem `app/abilities/ability.rb`.

**Background jobs:** Solid Queue (DB-backed, không cần Redis).

**Email preview:** LetterOpener tại `/letter_opener` trong development.

**Models planned (specs/Roadmap.md):** Theo thứ tự phase — xem `specs/ERD.md` để biết schema đầy đủ.

## Testing conventions

- Framework: Minitest with FactoryBot, Faker, Shoulda Matchers, and Capybara/Selenium for system tests.
- `ActionDispatch::IntegrationTest` includes `Devise::Test::IntegrationHelpers` — use `sign_in user` in controller/integration tests.
- Tests run in parallel by default (`parallelize(workers: :number_of_processors)`).
- Fixtures are loaded globally; prefer factories (`create(:user)`) over fixtures for new tests.
- System tests are optional in CI (commented out in `config/ci.rb`).

## Code style

RuboCop uses `rubocop-rails-omakase` (Basecamp's opinionated defaults). Pre-commit hook via Overcommit runs RuboCop on every commit and fails on warnings. Run `bin/rubocop` before committing.

Annotated schema comments are auto-generated by `annotaterb` — do not edit header comments in model files manually.

## Soft delete

`discard` gem is included — use `include Discard::Model` on models that need soft-delete. Do not use `dependent: :destroy` where soft-delete is expected.

## Pagination & search

Use `pagy` for pagination and `ransack` for search/filtering in index actions.
